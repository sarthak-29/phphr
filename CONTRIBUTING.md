CONTRIBUTING TO PHPHR

Thank you for considering contributing to PHPHR Open Source.
We welcome bug reports, feature suggestions, documentation improvements,
and code contributions.

--------------------------------------------------

HOW YOU CAN CONTRIBUTE

You can help improve PHPHR in many ways:

- Reporting bugs
- Suggesting new features
- Improving documentation
- Fixing bugs
- Adding new features

--------------------------------------------------

REPORTING BUGS

Before submitting a bug report:
- Check existing issues to avoid duplicates
- Provide clear steps to reproduce the issue
- Include screenshots or error messages if possible

--------------------------------------------------

SUBMITTING FEATURE REQUESTS

Feature requests should include:
- Clear problem statement
- Proposed solution
- Business or technical benefit

--------------------------------------------------

CODE CONTRIBUTIONS

1. Fork the repository
2. Create a new branch
   git checkout -b feature/your-feature-name
3. Make your changes
4. Commit with a clear message
   git commit -m "Add feature: employee export"
5. Push your branch and create a Pull Request

--------------------------------------------------

CODING STANDARDS

- Follow PSR-12 PHP coding standards
- Keep code clean and well-documented
- Test your changes before submitting
